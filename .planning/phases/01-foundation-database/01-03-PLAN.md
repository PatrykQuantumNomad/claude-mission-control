If `routeTree.gen.ts` is NOT generated, the plugin order is wrong (RESEARCH.md Pitfall 3) — Task 1's vite.config.ts is broken and must be fixed before this task can complete.

If the build fails on TypeScript errors related to routeTree.gen, that's expected on the FIRST build (the file doesn't exist yet). Solution: run `npx tsc --noEmit --project tsconfig.json` AFTER `npm run dev` has generated the file once, OR add a placeholder. The cleanest fix is to set `"skipLibCheck": true` (already in tsconfig.json) and either run `npm run dev` once to seed routeTree, or add `frontend/src/routeTree.gen.ts` as a placeholder that the plugin overwrites:

```ts
// Placeholder; tanstackRouter plugin overwrites this on first dev/build run.
export const routeTree = {} as never
```

Adding the placeholder is the lower-friction option. Use it.
