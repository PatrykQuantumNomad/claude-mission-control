## Decisions Made

- **Sub-step gating** — Task 1 split into 1A (8 simpler tables) and 1B (7 complex). After 1A: metadata-count + spot-check 3 tables; after 1B: final metadata-count + spot-check 3 complex tables. Gate caught zero issues this run, but the structure protects against Pitfall 2 (forgotten import in __init__.py producing a partial autogen migration).
- **`sa_column=Column(... ForeignKey ondelete=...)` for FKs with ON DELETE behavior** — SQLModel's `Field(foreign_key="parent.col")` shorthand omits `ondelete`. To express ON DELETE CASCADE / SET NULL the column must be declared via `sa_column=Column(...)`. Used uniformly across tools/live_state (CASCADE) and otel_events/decisions/inbox/tasks (SET NULL).
- **Free-text enums** — All [NEEDS USER CONFIRMATION] enum columns (sessions.source, sessions.outcome, tasks.quadrant, live_state.state, skills.environment) accepted as free-text per the APPROVED 2026-04-25 schema. A future migration can add CHECK constraints once production-dashboard reality clarifies the discrete enum values.
- **`tasks.skill` and `schedules.skill` as free-text** — Per the SCHEMA flag, neither column declares an enforced FK to skills.name. Soft-reference avoids breaking when a skill is renamed/deleted mid-flight.
- **Manual `import sqlmodel`** — Autogenerate references `sqlmodel.sql.sqltypes.AutoString` throughout the migration but never emits `import sqlmodel`. Without it, the migration raises NameError on first run. Pitfall 5 mitigated by hand-editing the autogenerate output.
- **Stable revision id `0001_initial`** — Replaces the autogenerated `d18b32ad99e5`. Predictable id keeps test fixtures, downgrade targets, and migration listings stable across regenerations.
- **`_column_exists` placed inside 0001_initial.py** — Per plan instruction. Future revisions either import via `importlib.util.spec_from_file_location("_initial", "migrations/versions/0001_initial.py")` or copy-paste the function body (no module state). Plan 06 / Phase 2+ may extract to a `migrations/_helpers.py` shared module if cross-revision import friction emerges.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking issue] alembic CLI URL resolves relative to cwd, `data/` did not exist under `backend/`**

- **Found during:** Task 2 first attempt at `alembic revision --autogenerate`.
- **Issue:** alembic.ini's `sqlalchemy.url = sqlite+aiosqlite:///data/cmc.db` is a relative path that resolves to `<cwd>/data/cmc.db`. When running from `backend/`, `backend/data/` did not exist (it was never created — Plan 04 deferred this to Plan 06's lifespan), so aiosqlite raised `OperationalError: unable to open database file`. The autogenerate command needs to actually open a connection (to compare against), so this blocked the task.
- **Fix:** `mkdir -p backend/data/` to satisfy the relative URL during autogenerate. This directory is gitignored (`data/*.db`, `data/*.db-wal`, `data/*.db-shm` already in `.gitignore`), so no DB artifacts are committed. The directory itself remains untracked and is created on demand at app startup by Plan 06's lifespan; this `mkdir` is purely a Plan 05 development convenience.
- **Files modified:** none (created `backend/data/` directory only; no tracked files).
- **Verification:** `alembic --raiseerr revision --autogenerate -m "..."` ran cleanly and produced the migration.
- **Committed in:** N/A (no tracked file change).

**2. [Rule 1 - Bug] Autogenerated migration references `sqlmodel.sql.sqltypes.AutoString` but never emits `import sqlmodel`**

- **Found during:** Task 2 hand-verification of autogenerated output.
- **Issue:** Every str column in the autogen output is rendered as `sqlmodel.sql.sqltypes.AutoString()`. Without `import sqlmodel` at the top, `alembic upgrade head` would raise `NameError: name 'sqlmodel' is not defined`. The default sqlmodel autogenerate template omits this import.
- **Fix:** Added `import sqlmodel` to the import block of 0001_initial.py during the hand-finalize pass.
- **Files modified:** `backend/migrations/versions/0001_initial.py` (manual import addition during initial write).
- **Verification:** Direct test run — `alembic upgrade head` against `/tmp/cmc_test_p05.db` succeeded and produced 16 tables (15 app + alembic_version).
- **Committed in:** `308c123` (Task 2 commit, with the import included from the start).

4. Each request reads `app.state.sessions` via `Depends(get_session)` (Plan 04).
