<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend response → frontend type | Untrusted JSON parsed into typed interface. Mitigation: `cost_usd: string` keeps the Decimal-as-string contract intact (parsing as number would lose precision); fetcher does not validate at runtime — relies on the backend Plan 01 schema contract. |
| URL path param (skill name) | User-provided to fetchers via panel state. Mitigation: encodeURIComponent in the fetcher PLUS server-side _SKILL_NAME_RE rejection. Defense-in-depth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Tampering | Skill name interpolated into URL path | mitigate | encodeURIComponent in fetcher + server-side regex validation. |
| T-14-02-02 | Information Disclosure | TanStack Query devtools shows query keys including skill names | accept | Single-user tool by design; devtools only available in dev mode. |
| T-14-02-03 | Tampering | Decimal cost_usd parsed as Number could lose precision | mitigate | TS interface declares `cost_usd: string`; downstream renderers must NOT call Number() on it. Code review gate (no automated test) — call out in Plan 03 panel implementations. |
