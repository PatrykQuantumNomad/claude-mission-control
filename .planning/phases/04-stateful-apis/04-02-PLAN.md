       @router.get("/decisions", response_model=DecisionListResponse)
       async def list_decisions(
           db: AsyncSession = Depends(get_session),
           status_: Optional[Literal["pending","answered"]] = Query(None, alias="status"),
           limit: int = Query(50, ge=1, le=200),
           offset: int = Query(0, ge=0),
       ):
           q = select(Decision)

    2. POST /decisions (HITL-02) — INSERT OR IGNORE on partial unique index:

       Use FastAPI's `Response` parameter (status_code default 201; reset to 200
       on the conflict path):
       ```python
       @router.post("/decisions", response_model=DecisionListItem, status_code=201)
       async def create_decision(
           payload: DecisionCreate,
           response: Response,
           db: AsyncSession = Depends(get_session),
       ):
           stmt = (
               sqlite_insert(Decision)
               .values(
                   session_id=payload.session_id,
                   task_id=payload.task_id,
                   dedup_key=payload.dedup_key,
                   prompt=payload.prompt,
                   options=payload.options,
                   status="pending",
                   created_at=datetime.now(timezone.utc),
               )
               .on_conflict_do_nothing(
                   index_elements=["dedup_key"],
                   index_where=text("status = 'pending'"),
               )
               .returning(Decision)
           )
           result = await db.execute(stmt)
           row = result.scalar_one_or_none()
           if row is None:
               # Conflict: fetch the existing pending row (Pitfall 6).
               existing = (await db.execute(
                   select(Decision).where(
                       Decision.dedup_key == payload.dedup_key,
                       Decision.status == "pending",
                   )
               )).scalar_one()
               await db.commit()
               response.status_code = 200
               return DecisionListItem.model_validate(existing)
           await db.commit()
           return DecisionListItem.model_validate(row)
       ```

    3. POST /decisions/{decision_id}/answer (HITL-03):
       ```python
       @router.post("/decisions/{decision_id}/answer", response_model=DecisionAnswerResponse)
       async def answer_decision(
           decision_id: int,
           payload: DecisionAnswerRequest,
           db: AsyncSession = Depends(get_session),
       ):
           row = (await db.execute(
               select(Decision).where(Decision.id == decision_id)
           )).scalar_one_or_none()
           if row is None:
               raise HTTPException(status_code=404, detail="decision not found")
           if row.status == "answered":
               raise HTTPException(status_code=409, detail="decision already answered")
           # File FIRST (Pitfall 1) — if this raises, no DB UPDATE happens.
           queue_path = write_decision_answer(
               decision_id, payload.answer, payload.answered_by,
           )
           now = datetime.now(timezone.utc)
           row.status = "answered"
           row.answer = payload.answer
           row.answered_at = now
           row.answered_by = payload.answered_by
           await db.commit()
           return DecisionAnswerResponse(
               answered=True, decision_id=decision_id, queue_path=str(queue_path),
           )
       ```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | T (Tampering) | Path traversal via decision_id / inbox_id | mitigate | Pydantic Field types in path operation are int; queue_path() cast int(decision_id) defensive int-coercion in cmc.core.queue.write_decision_answer / write_inbox_reply (Wave 0) |
| T-04-02-02 | I (Information Disclosure) | IntegrityError leakage on HITL-02 | mitigate | INSERT OR IGNORE pattern silently suppresses constraint violations; conflict path returns the existing pending row (200) instead of bubbling the SQLite error |
| T-04-02-03 | T (Tampering) | Crash window between queue file write and DB UPDATE | mitigate | File write FIRST, then DB UPDATE (Pitfall 1). If the file fails, no DB change → dispatcher can resend safely. The reverse order is the bug pattern. |
| T-04-02-04 | D (DoS) | Unbounded inbox/decision body size | mitigate | Pydantic Field max_length on prompt (10000), body (10000), reply (10000), answer (10000) — caps memory + queue file growth. |
