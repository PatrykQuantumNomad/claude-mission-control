**Option B for token_usage (research §4 + Open Question 4 + planning_context):**
  When re-parsing a session whose previous totals already counted toward a daily
  bucket, the rollup must NOT double-count. Strategy:
    1. Before parsing the file, fetch existing session row's per-day token contribution
       (we approximate: existing session row's totals are used as a single rollup
       contribution; if the session spans multiple days we accept slight smearing —
       the parser-emitted `token_usage_buckets` are per (day, model) and their SUM
       equals session totals, so we can do a proportional split on re-parse).
    2. Simpler approach (chosen): the `token_usage_buckets` returned by the parser
       are KEYED BY (day, model). For each new bucket, if the previous session row
       had non-zero tokens, we cannot trivially split them by day post-hoc. Instead:
         - Track a per-session `previous_buckets` table? Heavier than what Phase 2
           needs.
         - Use this Phase 2 simplification: on re-parse, DELETE all existing
           token_usage row's contributions for THIS session by storing a reverse
           bucket and applying it. Phase 2 chooses the SIMPLEST viable Option B
           variant: store previous bucket totals in a tiny scratch table OR
           recompute the FULL per-session contribution each cycle.

  PHASE 2 LOCKED VARIANT: don't introduce a scratch table. Instead, compute the
  net delta per (day, model) per sync cycle using a simple invariant:
    - Each parse re-derives the FULL per-(day,model) contribution of THIS session.
    - We store on the `sessions` row enough info to know what was previously contributed:
      we already have `tokens_input/output/cache_read/cache_create` per session BUT NOT
      per (day, model). For the Phase 2 v1: ONE-DAY HEURISTIC.
    - Phase 2 v1 simplification: a single session's tokens are bucketed under the
      LATEST (day, model) the session reached at sync time. On re-parse: subtract
      the existing session row's totals from that (day, model) bucket, then add
      the new totals to the new latest bucket. This is acceptable for v1 because
      virtually all sessions complete within one local day (research note in §6).
    - Multi-day sessions: tokens land in the day the LATEST assistant message
      occurred (slight smear; documented as known limitation; revisit in Phase 3+).

  IMPLEMENTATION OF Option B (Phase 2 v1):
    Step 1: get_existing_session_for_path(db, jsonl_path) -> Session | None
    Step 2: If existing: subtract existing.tokens_* from token_usage row keyed by
            (existing.last_day, existing.model). Compute existing.last_day from
            existing.synced_at converted to local date (close enough; see caveat).
    Step 3: Run upsert_session with new totals.
    Step 4: For each parsed bucket: insert new contribution (Option B add).

  CAVEAT (documented in summary): "last_day for subtraction is approximated from
  sessions.synced_at converted to local date. For sessions that span midnight this
  produces small drift on re-parse. Revisit in Phase 3+ if needed."

  ALTERNATIVE (simpler): Phase 2 v1 ACCEPTS the simplification: re-parse cycles
  are 120s apart, so within one parse-cycle window the rollup is at most 1 cycle
  out of date. Tests will validate the no-double-count invariant on simple
  (single-day, single-model) flows; multi-day gets a known-limitation note.

Implement `backend/cmc/ingest/repository.py`. All functions take an `AsyncSession` and DO NOT commit (caller commits per-cycle to keep transactions atomic). Use `from sqlalchemy.dialects.sqlite import insert` for upsert.

```python
"""Idempotent SQLAlchemy upsert helpers for Phase 2 ingestion.

Per research §4 + Open Question 4 (locked: Option B):
  - Sessions: upsert on PK session_id; immutable fields (started_at) are NOT
    overwritten; mutable fields (totals, ended_at, synced_at, jsonl_mtime) are.
  - Tools: upsert on the unique constraint tool_use_id; pending → ok/error
    transition without creating a duplicate.
  - Token usage: Option B subtract-then-add per (day, model, source). Each call
    accepts the SESSION's previous totals (or None on first parse) so re-parses
    don't double-count.

All functions DO NOT commit. The caller wraps the unit-of-work in a single commit.
"""
from __future__ import annotations

from datetime import date as _date, datetime
from pathlib import Path
from typing import Sequence

from sqlalchemy import select, update
from sqlalchemy.dialects.sqlite import insert
from sqlalchemy.ext.asyncio import AsyncSession

from cmc.db.models.sessions import Session as SessionModel
from cmc.db.models.tools import ToolCall
from cmc.db.models.token_usage import TokenUsage


async def get_existing_session_for_path(db: AsyncSession, jsonl_path: Path) -> SessionModel | None:
    """Look up an existing session by jsonl_path. Used by sync_once to decide
    whether to skip (no mtime change) or re-parse (mtime newer)."""
    stmt = select(SessionModel).where(SessionModel.jsonl_path == str(jsonl_path)).limit(1)
    result = await db.execute(stmt)
    return result.scalar_one_or_none()


async def upsert_session(db: AsyncSession, **fields) -> None:
    """Insert or update a sessions row keyed on session_id.

    Mutable cols are taken from `excluded.*`; immutable started_at is NOT
    overwritten by ON CONFLICT.
    """
    stmt = insert(SessionModel).values(**fields)
    update_cols = {
        c: getattr(stmt.excluded, c) for c in (
            "ended_at", "synced_at", "jsonl_mtime", "jsonl_path",
            "cwd", "project_hash", "model", "source", "outcome",
            "tokens_input", "tokens_output", "tokens_cache_read",
            "tokens_cache_create", "tool_call_count", "message_count",
            "error_message",
        )
    }
    stmt = stmt.on_conflict_do_update(index_elements=["session_id"], set_=update_cols)
    await db.execute(stmt)


async def upsert_tools(
    db: AsyncSession, session_id: str, tool_calls: Sequence[dict]
) -> None:
    """Upsert each tool call on the unique tool_use_id constraint.

    Pending → ok/error transitions update the existing row in place (no dup).
    """
    for tc in tool_calls:
        row = {
            "session_id": session_id,
            "tool_use_id": tc["tool_use_id"],
            "tool_name": tc["tool_name"],
            "started_at": tc["started_at"],
            "ended_at": tc.get("ended_at"),
            "duration_ms": tc.get("duration_ms"),
            "status": tc["status"],
            "mcp_server_name": tc.get("mcp_server_name"),
            "mcp_tool_name": tc.get("mcp_tool_name"),
            "input_summary": tc.get("input_summary"),
        }
        stmt = insert(ToolCall).values(**row)
        stmt = stmt.on_conflict_do_update(
            index_elements=["tool_use_id"],
            set_={
                "ended_at": stmt.excluded.ended_at,
                "duration_ms": stmt.excluded.duration_ms,
                "status": stmt.excluded.status,
                "mcp_server_name": stmt.excluded.mcp_server_name,
                "mcp_tool_name": stmt.excluded.mcp_tool_name,
                "input_summary": stmt.excluded.input_summary,
            },
        )
        await db.execute(stmt)


async def accumulate_token_usage(
    db: AsyncSession,
    session_id: str,
    previous_totals: dict | None,
    new_buckets: Sequence[dict],
    primary_day: _date | None,
    primary_model: str | None,
) -> None:
    """Apply Option B subtract-then-add per (day, model, source) for token_usage.

    `previous_totals` is None on first parse; otherwise a dict with keys
    tokens_input/output/cache_read/cache_create representing what the existing
    session ALREADY contributed to the rollup.

    `primary_day` + `primary_model` identify the (day, model, source) bucket
    that should have the previous_totals subtracted (Phase 2 v1 simplification:
    we attribute previous totals to the latest known (day, model) of the session).
    For first parse this argument pair is unused (None).

    For each new bucket, ADD the deltas to the corresponding (day, model, source)
    row; create the row if missing.
    """
    # Step 1: subtract previous contribution from primary bucket, if applicable.
    if previous_totals and primary_day and primary_model:
        await _adjust_bucket(
            db,
            day=primary_day, model=primary_model, source="claude-code",
            tokens_input=-int(previous_totals.get("tokens_input", 0)),
            tokens_output=-int(previous_totals.get("tokens_output", 0)),
            tokens_cache_read=-int(previous_totals.get("tokens_cache_read", 0)),
            tokens_cache_create=-int(previous_totals.get("tokens_cache_create", 0)),
            sessions_count_delta=0,  # session count unchanged on re-parse
        )

    # Step 2: add new buckets.
    is_new_session = previous_totals is None
    for b in new_buckets:
        await _adjust_bucket(
            db,
            day=b["day"], model=b["model"], source=b.get("source") or "claude-code",
            tokens_input=int(b.get("tokens_input", 0)),
            tokens_output=int(b.get("tokens_output", 0)),
            tokens_cache_read=int(b.get("tokens_cache_read", 0)),
            tokens_cache_create=int(b.get("tokens_cache_create", 0)),
            sessions_count_delta=1 if is_new_session else 0,
        )


async def _adjust_bucket(
    db: AsyncSession, *,
    day: _date, model: str, source: str,
    tokens_input: int, tokens_output: int,
    tokens_cache_read: int, tokens_cache_create: int,
    sessions_count_delta: int,
) -> None:
    """Insert or update a token_usage row, ADDING the deltas to the existing values.

    Negative deltas are valid (used by Option B subtract step).
    Uses an UPDATE-then-INSERT-fallback pattern because SQLAlchemy's
    on_conflict_do_update with arithmetic on excluded.* + existing column is
    awkward to express portably. SQLite only requires one round-trip per row,
    which is fine at Phase 2 scale.
    """
    # Try to update first
    upd = (
        update(TokenUsage)
        .where(
            (TokenUsage.day == day)
            & (TokenUsage.model == model)
            & (TokenUsage.source == source)
        )
        .values(
            tokens_input=TokenUsage.tokens_input + tokens_input,
            tokens_output=TokenUsage.tokens_output + tokens_output,
            tokens_cache_read=TokenUsage.tokens_cache_read + tokens_cache_read,
            tokens_cache_create=TokenUsage.tokens_cache_create + tokens_cache_create,
            sessions_count=TokenUsage.sessions_count + sessions_count_delta,
            updated_at=datetime.utcnow(),
        )
    )
    result = await db.execute(upd)
    if result.rowcount and result.rowcount > 0:
        return
    # Row didn't exist: insert it.
    stmt = insert(TokenUsage).values(
        day=day, model=model, source=source,
        tokens_input=tokens_input, tokens_output=tokens_output,
        tokens_cache_read=tokens_cache_read, tokens_cache_create=tokens_cache_create,
        sessions_count=sessions_count_delta,
        updated_at=datetime.utcnow(),
    )
    # In a race where another inserter wins, on_conflict_do_update applies the same
    # delta math via column arithmetic. Use NoNothing for simplicity here since
    # this codepath is single-writer per Phase 2 design.
    stmt = stmt.on_conflict_do_update(
        index_elements=["day", "model", "source"],
        set_={
            "tokens_input": TokenUsage.tokens_input + stmt.excluded.tokens_input,
            "tokens_output": TokenUsage.tokens_output + stmt.excluded.tokens_output,
            "tokens_cache_read": TokenUsage.tokens_cache_read + stmt.excluded.tokens_cache_read,
            "tokens_cache_create": TokenUsage.tokens_cache_create + stmt.excluded.tokens_cache_create,
            "sessions_count": TokenUsage.sessions_count + stmt.excluded.sessions_count,
            "updated_at": stmt.excluded.updated_at,
        },
    )
    await db.execute(stmt)
```

```python
"""Periodic JSONL sync orchestration.

Phase 2 contract:
  - sync_once(sessionmaker, settings) is the unit-of-work used both by
    cmc.app.lifespan (boot-time + periodic) and cmc.api.routes.sync (manual /api/sync).
  - periodic_sync_loop(sessionmaker, settings, interval_s=120) wraps sync_once
    in a never-ending while True loop with cancellation hygiene per research
    §3 + Pitfall 7.
"""
from __future__ import annotations

import asyncio
import logging
from datetime import datetime, timedelta, timezone
from pathlib import Path

from sqlalchemy.ext.asyncio import async_sessionmaker

from cmc.config import Settings
from cmc.ingest.jsonl_parser import parse_session_file
from cmc.ingest.repository import (
    accumulate_token_usage, get_existing_session_for_path,
    upsert_session, upsert_tools,
)

log = logging.getLogger(__name__)


async def sync_once(sessionmaker: async_sessionmaker, settings: Settings) -> dict:
    """One full ingestion cycle. Returns a summary dict for logs / API response."""
    started_clock = asyncio.get_event_loop().time()
    summary: dict = {"files_seen": 0, "files_updated": 0, "errors": 0}

    root = Path(settings.jsonl_root).expanduser()
    if not root.is_dir():
        log.warning("ingest.jsonl_root_missing path=%s", root)
        summary["duration_ms"] = 0
        return summary

    # ONE LEVEL ONLY — never **/*.jsonl (would scoop subagents per Pitfall 5).
    for jsonl_path in sorted(root.glob("*/*.jsonl")):
        summary["files_seen"] += 1
        try:
            await _sync_one_file(sessionmaker, settings, jsonl_path, summary)
        except Exception:
            log.exception("ingest.file_error path=%s", jsonl_path)
            summary["errors"] += 1

    summary["duration_ms"] = int((asyncio.get_event_loop().time() - started_clock) * 1000)
    return summary


async def _sync_one_file(
    sessionmaker: async_sessionmaker,
    settings: Settings,
    jsonl_path: Path,
    summary: dict,
) -> None:
    mtime = datetime.fromtimestamp(jsonl_path.stat().st_mtime)
    async with sessionmaker() as db:
        existing = await get_existing_session_for_path(db, jsonl_path)
        # Skip if existing session is already ended AND mtime didn't move forward.
        if existing and existing.ended_at and existing.jsonl_mtime and existing.jsonl_mtime >= mtime:
            return

        # Heavy parsing happens off the event loop (research §5 / Pitfall 5).
        parsed = await asyncio.to_thread(parse_session_file, jsonl_path)
        sess = parsed["session"]
        if not sess.get("session_id"):
            return  # empty / unparseable

        # ended_at decision via mtime heuristic (research §1).
        last_ts = sess.pop("_last_message_ts", None)
        idle = timedelta(minutes=settings.session_idle_minutes)
        ended_at = last_ts if (datetime.now() - mtime) > idle else None

        sess["jsonl_path"] = str(jsonl_path)
        sess["jsonl_mtime"] = mtime
        sess["synced_at"] = datetime.utcnow()
        sess["source"] = "claude-code"
        sess["project_hash"] = jsonl_path.parent.name
        sess["ended_at"] = ended_at

        # Compute Option B previous-totals BEFORE upsert (so we still have the
        # existing row's old values).
        previous_totals = None
        primary_day = None
        primary_model = None
        if existing:
            previous_totals = {
                "tokens_input": existing.tokens_input or 0,
                "tokens_output": existing.tokens_output or 0,
                "tokens_cache_read": existing.tokens_cache_read or 0,
                "tokens_cache_create": existing.tokens_cache_create or 0,
            }
            # Phase 2 v1 simplification: attribute previous-totals to the latest
            # (day, model) the session was last synced under (research note).
            primary_day = (existing.synced_at or datetime.utcnow()).astimezone().date()
            primary_model = existing.model or "unknown"

        await upsert_session(db, **sess)
        await upsert_tools(db, sess["session_id"], parsed["tool_calls"])
        await accumulate_token_usage(
            db,
            session_id=sess["session_id"],
            previous_totals=previous_totals,
            new_buckets=parsed["token_usage_buckets"],
            primary_day=primary_day,
            primary_model=primary_model,
        )
        await db.commit()
        summary["files_updated"] += 1


async def periodic_sync_loop(
    sessionmaker: async_sessionmaker,
    settings: Settings,
    interval_s: int = 120,
) -> None:
    """Sleep then sync, forever. Catches Exception, re-raises CancelledError.

    Pitfall 7: bare `except Exception` (NOT BaseException) so cancellation
    propagates while transient errors (DB lock, FS hiccup) don't kill the loop.
    """
    while True:
        try:
            await asyncio.sleep(interval_s)
            summary = await sync_once(sessionmaker, settings)
            log.info("ingest.cycle %s", summary)
        except asyncio.CancelledError:
            log.info("ingest.cycle_cancelled")
            raise
        except Exception:
            log.exception("ingest.cycle_unexpected_error")
            continue
```
