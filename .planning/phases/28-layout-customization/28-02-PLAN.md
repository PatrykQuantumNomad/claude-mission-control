Create `frontend/src/lib/layout/useLayoutState.ts` NEW per RESEARCH.md "Read URL → derive layout state" example + §7 reset mechanics.

Hook signature: `useLayoutState(route: string)` returns `{ isHidden, setHidden, orderedPanels, setOrder, splitSizes, setSplit, reset }`.

Dependencies:
- `useNavigate` + `useRouterState` from `@tanstack/react-router`
- `useMemo` for derived sets/maps
- `PANEL_REGISTRY` + `isValidPanelId` from `./panelRegistry`

Implementation guidance:

`useRouterState({ select: (s) => s.location.search })` returns the validated search object. Cast as `Record<string, unknown>` per Phase 26 pattern.

`isHidden(panelId)` reads hidden_panels CSV into a Set memoized on the CSV string. Membership test — that's it. Unknown panel ids (not in registry) silently pass through the Set; the route's render code calls `isValidPanelId` before consulting `isHidden` (Pitfall 7 defense in depth).

`setHidden(panelId, hide)` computes next Set, serializes to sorted CSV (sort for deterministic fork-save round-trip per Phase 26 Plan 07 precedent), and navigates `{ to: '.', search: prev => ({ ...prev, hidden_panels: csv || undefined }), replace: true }`. When the Set is empty, pass `undefined` not '' (Pitfall 2). `replace: true` prevents back-button noise (Phase 25/26 pattern).

`orderedPanels(columnId)`: read panel_order CSV, parse into Record<columnId, panelId[]>. For the requested columnId: take the URL-specified ids (filter unknown via PANEL_REGISTRY[route] membership), then append any registry-known ids for this column that the URL did NOT mention, in registry order. Result is "URL drives order for known panels; new panels added since the URL was minted appear at the end" (Pitfall 7 graceful drift).

`setOrder(columnId, panelIds)`: serialize the per-column array back into CSV groups joined by `;`. Preserve other columns' orders from the current URL. Navigate with new panel_order param.

`splitSizes(groupId)`: read split_sizes CSV, parse into Record<groupId, number[]>. Return the array for groupId or undefined.

`setSplit(groupId, sizes)`: accepts `number[] | null`. When `sizes` is a non-empty array: round each value to integer (RESEARCH.md §1 — "Round to 1-decimal to keep URL short" → use integer for simplicity in v1), preserve other groups' sizes, serialize to CSV groups joined by `;`, and navigate. When `sizes` is `null` OR an empty array: PRUNE the named group from the current split_sizes map; if the resulting map has zero groups, pass `undefined` for split_sizes (param removed entirely — NEVER empty string per Pitfall 2); otherwise serialize the remaining groups and navigate. This null/empty-array overload is consumed by Plan 28-05's ResizablePanelGroup double-click-reset handler so the URL drops `split_sizes` when the user resets back to defaults (LAYO-03 SC#2).

`reset()`: navigate with `search: prev => { const next = { ...prev }; delete next.hidden_panels; delete next.panel_order; delete next.split_sizes; return next }`. CRITICAL: do NOT touch any other keys. The current code path's stableness across `time_from` / `time_to` / `compare_panels` / `range` / `a` / `b` / `schemaVersion` is the LAYO-04 SC#3 lock. Use the destructuring-delete pattern, NOT a whitelist-spread, to be append-only-safe against future search keys.

Optimization: `reset` should be a no-op if the URL currently has none of the three layout keys (avoid history pollution from accidentally clicking reset when nothing is customized). Detect via `if (!search.hidden_panels && !search.panel_order && !search.split_sizes) return;`.

Stable identity: wrap `setHidden`/`setOrder`/`setSplit`/`reset` in `useCallback` keyed on the relevant search-param dependencies. `isHidden` / `orderedPanels` / `splitSizes` can be inlined or useCallback'd — Wave 3 perf gate (Pitfall 6) requires no extraneous render churn.
