# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5 (Frontend Shell & Design System). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Locked decisions trace to REQUIREMENTS.md (DESG-01..06, FESH-01..10) and the canonical palette in `build-your-own-dashboard-guide.html`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | manual (CSS custom properties + utility classes; no shadcn CLI) |
| Preset | not applicable |
| Component library | bespoke primitives in `frontend/src/components/ui/` styled via CSS variables (no Tailwind, no Radix shipped — shadcn-style copy-in primitives may be hand-translated where useful: e.g. Sheet patterns from Radix Dialog if installed for FESH-04, Tooltip from Radix Tooltip if installed for FESH-06) |
| Icon library | `lucide-react` (DESG-06 — locked) |
| Font | Inter (body, headings — DESG-03 locked); JetBrains Mono (labels, kickers, numeric displays, timestamps — DESG-03 locked); both loaded via Google Fonts CDN (`https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=JetBrains+Mono:wght@400;500;600&display=swap`) |

**Why not shadcn CLI:** REQUIREMENTS.md locks bespoke design tokens (`#0a0a0f` bg, `#12121a` surface, `#4d7cff → #8b5cf6` accent gradient, custom radial backgrounds DESG-02, 14–16px radius DESG-04) that do not map cleanly to any shadcn preset's neutral/zinc/slate palette. Forcing a preset would generate tokens the executor must immediately overwrite. The contract instead defines explicit CSS custom properties in `frontend/src/styles.css` (extending the existing Phase 1 stub at lines 1–10) and ships hand-rolled primitives that already match the spec.

**Optional Radix dependency:** Phase 5 planner may add `@radix-ui/react-dialog` (Sheet — FESH-04 modal/focus-trap/Esc behavior), `@radix-ui/react-tooltip` (FESH-06 + DESG-03 hover timestamps), `@radix-ui/react-collapsible` (FESH-03) if hand-rolling these primitives proves out of scope. These provide unstyled accessibility primitives only — visual styling stays bespoke per this contract. `framer-motion` is REQUIRED (FESH-03 — 220ms height animation; DESG-05 — 300ms panel fade-in).

## Spacing Scale

Declared values (all multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 4px | Icon gaps, badge inline padding, dot-divider spacing |
| xs | 8px | Compact element spacing, small badge padding |
| sm | 12px | Tight component internal padding (compact-input slot — see Exceptions) |
| md | 16px | Default element spacing, header padding (DESG-04 — already in Phase 1 styles.css) |
| lg | 24px | Section padding, page main padding (DESG-04 — already in Phase 1 styles.css), card padding lower bound |
| xl | 32px | Layout gaps, card padding upper bound (DESG-04 declares 24–32px card padding) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top spacing for hero/empty states |

Border-radius scale (DESG-04 — 14–16px primary; corroborated by build-your-own-dashboard-guide.html):

| Token | Value | Usage |
|-------|-------|-------|
| radius-sm | 8px | Badges, status pills (StatePill), small inputs |
| radius-md | 12px | Buttons, ghost buttons, secondary controls |
| radius-lg | 14px | Cards (default — DESG-04), Sheet header/footer |
| radius-xl | 16px | Large cards, modal panels, command palette container |

Exceptions:
- **`--space-sm: 12px` (off-standard, approved exception):** Used for compact input internal vertical padding (CommandPalette query input, header Cmd+K trigger chip, inline filter pills, NavBar link vertical padding). Rationale — 8px clips Inter 14px-body ascenders/descenders inside bordered inputs and breaks DESG-03's data-density target on the Command screen; 16px inflates input heights past the NavBar 56px header and fights the dense KPI strip below. 12px is the minimum value that preserves both legibility and density. Restricted to the listed surfaces — never used for component-to-component layout gaps (those use the standard set {4, 8, 16, 24, 32, 48, 64}).
- Avatar/icon circles use `border-radius: 50%` (StatePill dot indicators only — never on text containers)
- Skeleton placeholders inherit their parent's radius (FESH-08)
- Touch targets MUST be at least 32×32px hit area even when visual element is smaller (use padding to expand)

## Typography

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Body | 14px | 400 (regular) | 1.5 (= 21px) | Inter |
| Label/Kicker | 12px | 500 (inline only — see note) | 1.4 (= 16.8px) — uppercase, letter-spacing 0.05em | JetBrains Mono |
| Heading | 18px | 600 (semibold) | 1.3 (= 23.4px) | Inter |
| Display | 28px | 600 (semibold) | 1.2 (= 33.6px) | Inter |

Sizes: 4 (12, 14, 18, 28). Weights: 2 named tokens — `--weight-regular: 400` and `--weight-semibold: 600`. JetBrains Mono label/kicker uses `font-weight: 500` directly (not a named token) — applied inline only in the `.cmc-label` / kicker rule in `frontend/src/styles.css` because medium reads better than 600 at small caps in monospace (DESG-03 rationale). The 500 value is intentionally NOT exported as a CSS custom property to keep the design-token contract at exactly 2 weights.

**Numeric/data display rule:** All numeric values in panels (token counts, latency ms, durations, percentages, KPI tile big-numbers) use JetBrains Mono at the role-appropriate size. Tabular figures via `font-variant-numeric: tabular-nums`.

**Relative timestamps (FESH-10):** body text shows relative ("3m ago") in JetBrains Mono at 12px; absolute timestamp in tooltip on hover (uses Tooltip primitive — Radix or hand-rolled).

## Color

Source of truth: REQUIREMENTS.md DESG-01 + build-your-own-dashboard-guide.html `:root` block. All values become CSS custom properties on `:root` in `frontend/src/styles.css`.

| Role | CSS variable | Value | 60/30/10 bucket | Usage |
|------|--------------|-------|-----------------|-------|
| Page background | `--cmc-bg` | `#0a0a0f` | Dominant 60% | App body background, page-level containers (already wired in Phase 1) |
| Card surface | `--cmc-surface` | `#12121a` | Dominant 60% (extends bg) | Card backgrounds, sheet panels, modal panels |
| Elevated surface | `--cmc-surface-2` | `#1a1a27` | Secondary 30% | Hover states on cards, secondary buttons, sub-cards |
| Highest elevation | `--cmc-surface-3` | `#22222f` | Secondary 30% | Inline code blocks, expandable section headers, drawer footers |
| Border | `--cmc-border` | `#2a2a3d` | Secondary 30% | Card outlines, divider lines, input borders |
| Border (focus/glow) | `--cmc-border-glow` | `#3d3d5c` | Secondary 30% | Hover/focus ring on cards and buttons |
| Text primary | `--cmc-text` | `#e8e8f0` | Secondary 30% | Body, headings, primary content |
| Text dim | `--cmc-text-dim` | `#8888a0` | Secondary 30% | Secondary labels, supporting copy, captions |
| Text subtle | `--cmc-text-subtle` | `#5a5a70` | Secondary 30% | Tertiary metadata, placeholder text, kicker labels when not active |
| Accent (primary) | `--cmc-accent-blue` | `#4d7cff` | Accent 10% | Primary CTAs (gradient start), active nav, focus ring inner color |
| Accent (secondary) | `--cmc-accent-purple` | `#8b5cf6` | Accent 10% | Gradient end on primary CTA, command palette highlight, "answer required" pulse |
| Accent gradient | `--cmc-gradient-hero` | `linear-gradient(135deg, #4d7cff, #8b5cf6)` | Accent 10% | Primary button background, branded display text gradient (background-clip: text) |
| Status — success | `--cmc-status-green` | `#10b981` | Accent 10% (status only) | StatePill `ok`/`active`, schedule "next-run-on-time", success toast |
| Status — warning | `--cmc-status-amber` | `#f59e0b` | Accent 10% (status only) | StatePill `pending`/`stale`/`degraded`, attention bar warnings |
| Status — danger | `--cmc-status-red` | `#ef4444` | Accent 10% (status only) | StatePill `error`/`failed`, AttentionBar critical, EmergencyStop button |
| Status — info | `--cmc-status-cyan` | `#06b6d4` | Accent 10% (status only) | StatePill `running`/`live`, OTEL firehose event accent, live SSE indicators |

**Accent reserved for** (DESG-01 — explicit list, not "all interactive elements"):
1. Primary CTAs (Button variant=`primary` only — uses `--cmc-gradient-hero`)
2. Active state on top navigation links (current route — gradient underline 2px, text in `--cmc-accent-blue`)
3. Focus ring on keyboard-navigated elements (`outline: 2px solid --cmc-accent-blue`, `outline-offset: 2px`)
4. Command palette query input cursor + selected-item background tint (`rgba(77, 124, 255, 0.12)`)
5. Branded heading gradient on main page hero/title only (NOT body headings) — `background-clip: text`

Status colors are reserved EXCLUSIVELY for state communication (StatePill, AttentionBar, EmergencyStop). They never substitute for the accent gradient on neutral interactive elements.

**Background depth (DESG-02 — layered radial gradients):**
```css
body {
  background-color: var(--cmc-bg);
  background-image:
    radial-gradient(circle at 20% 0%, rgba(77, 124, 255, 0.05) 0%, transparent 50%),
    radial-gradient(circle at 80% 100%, rgba(139, 92, 246, 0.05) 0%, transparent 50%);
  background-attachment: fixed;
}
```

**Visual focal point (Command route `/`):** The primary focal point on the Command screen is the **KpiRow tile strip** — the first content element directly below the NavBar — which anchors the eye before the panel grid below. Hierarchy descends: KpiRow (display-size numerics in mono) → AttentionBar (status-amber/red when active) → Card grid (LiveSessions, RecentTasks, NextRuns) → footer skeletons. On Activity (`/activity`) the focal point is the OTEL event firehose head; on Skills (`/skills`) the focal point is the Skills directory grid header card. (Phase 5 ships only the shell — KpiRow and panel grid are Phase 6 deliverables — but the AppShell layout slot ordering is locked here so Phase 6 inherits the focal-point contract.)

**Destructive color:** `--cmc-status-red` (`#ef4444`). Used for: EmergencyStop banner button (HPNL/TPNL phases), Delete actions in Task board, "Fail task" confirm action. Never for general errors — those use status-red-tinted toast/banner without a confirm CTA.

## Copywriting Contract

This phase ships the SHELL — Phase 5 itself has no domain-specific CTAs (decisions, tasks, schedules land in Phases 6/7). The copy below covers shell-level surfaces ONLY.

| Element | Copy |
|---------|------|
| Primary CTA (CommandPalette quick-action — FESH-07) | "Quick task" (verb+noun; opens TaskComposer in Phase 7 — Phase 5 placeholder navigates to `/` with palette closed) |
| Empty state heading (any panel using shell primitives — FESH-09) | "Nothing to show yet" |
| Empty state body (default for shell-provided EmptyState component — FESH-09) | "Once {data-noun} arrives it will appear here. Run sync from the header to refresh." (where {data-noun} is panel-specific — passed as prop) |
| Empty state body (CommandPalette no-results) | "No matches. Try fewer letters or open the page directly." |
| Loading state (FESH-08 — skeletons not spinners) | NO copy — skeleton uses subtle pulse animation only; never "Loading…" text. Skeleton blocks match the final content shape. |
| Error state (shell-level fetch failure boundary) | Heading: "Couldn't reach the dashboard server." Body: "Check that `cc start` is running, then refresh. If this keeps happening, run `cc doctor` from your terminal." |
| Error state (network unavailable — offline) | Heading: "You're offline." Body: "The dashboard reconnects automatically. Last data shown was synced {relative-time}." |
| Destructive confirmation (none in Phase 5) | N/A — no destructive actions in Phase 5. EmergencyStop ships in TPNL-05 (Phase 7); Delete actions ship in Phase 7. The confirmation dialog component is a Phase 5 deliverable (a `ConfirmDialog` primitive in the shell library) but its copy is owned by the consuming panel. |
| Navigation labels (FESH-01 — three top-nav items) | "Command" (route `/`), "Activity" (route `/activity`), "Skills" (route `/skills`) — exact-case as shown |
| Header brand | "Mission Control" (drops "Claude" prefix used in Phase 1 stub `Claude Mission Control` to leave room for status pills + command palette trigger; the page `<title>` retains "Claude Mission Control") |
| Command palette trigger label | "Cmd+K" (kickers in JetBrains Mono inside a 1px border-radius=8px chip in the header — visible affordance per FESH-07) |
| Command palette placeholder | "Search pages, sessions, schedules…" |
| Theme indicator (DESG-01 — dark theme is the only theme; no toggle in v1) | None. Phase 9 TEST-04 references "theme toggle persists" but v1 ships dark-only; the test verifies the localStorage persistence pattern using CollapsibleSection state (FESH-03) as the canonical persistence carrier. |

**Voice rules:**
- Empty states explain WHY there's nothing AND what the user can do next (matches "teach the user what's happening" in FESH-09)
- Error states name the likely cause + the specific remediation command — never "Try again later"
- Sentence case throughout. Title case ONLY on top-nav labels and brand
- Never use "Loading…", "Fetching…", "Please wait…" text — use skeletons (FESH-08)
- Numbers > 999 use compact notation in copy ("12.4k tokens"), full digits in tables/dense displays

## Motion Contract (DESG-05)

| Surface | Property | Duration | Easing | Notes |
|---------|----------|----------|--------|-------|
| Panel/card mount | opacity 0→1, translateY 8px→0 | 300ms | `ease-out` | framer-motion `initial`/`animate`; staggered children at 50ms each |
| CollapsibleSection | height auto-toggle | 220ms | `ease-out` | FESH-03 explicit; framer-motion `<motion.div>` with `height: 'auto'` |
| Button hover | translateY 0→-2px + box-shadow grow | 150ms | `ease-out` | DESG-05 "hover lift 2px with shadow" |
| Sheet open | translateX 100%→0 (right side) + backdrop opacity 0→1 | 220ms | `ease-out` | Backdrop opacity caps at 0.6 |
| CommandPalette open | scale 0.96→1 + opacity 0→1 | 180ms | `ease-out` | Backdrop blur 8px |
| Tooltip open | opacity 0→1 + translateY 4px→0 | 150ms | `ease-out` | After 200ms hover delay |
| StatePill state change (ok→error) | background color tween | 300ms | `ease-in-out` | Avoids jarring instant color flips |

`prefers-reduced-motion: reduce` MUST disable all motion (set duration to 0ms, remove translate/scale, keep opacity for accessibility cues only).

## Accessibility Contract

| Concern | Requirement |
|---------|------------|
| Color contrast | Body text on bg ≥ 4.5:1 (`#e8e8f0` on `#0a0a0f` = ~14.6:1 ✓); dim text on surface ≥ 3:1 (`#8888a0` on `#12121a` = ~5.4:1 ✓); subtle text used ONLY for non-essential metadata |
| Focus visible | Always keyboard-visible — `:focus-visible` outline 2px `--cmc-accent-blue` with 2px offset, NEVER suppressed |
| Keyboard | Cmd+K opens palette (FESH-07); Esc closes Sheet/CommandPalette; Tab cycles within Sheet (focus trap — FESH-04); Enter triggers default button in dialogs |
| Screen reader | NavBar uses `<nav aria-label="Primary">`; CollapsibleSection emits `aria-expanded` + `aria-controls`; Sheet uses `aria-modal="true"` + labeled-by; Skeleton uses `aria-busy="true"` on parent + `aria-label="Loading"` on first skeleton block only |
| Reduced motion | `prefers-reduced-motion: reduce` cuts all transforms/translates; opacity-only fallbacks |
| Hit area | Interactive elements ≥ 32×32px hit area (use padding even when visual is smaller) |
