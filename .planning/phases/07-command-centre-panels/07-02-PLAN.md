## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API (POST /api/decisions/{id}/answer) | User-supplied answer text written to JSONL queue server-side; backend redacts nothing (answer is intended user content) |
| Browser → API (POST /api/inbox/{id}/reply) | Same shape |
| Browser → API (PATCH /api/skills/{name}/autonomy) | Path-bound name; backend regex + dotdot defense (already shipped Plan 03-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Repudiation | Decisions answer race (two clients answering same decision) | mitigate | Backend HITL-02 INSERT OR IGNORE returns 409 on dup; useAnswerDecision is NOT optimistic (Pitfall 2); user input preserved on rollback so they can re-read the body literal "decision already answered" and act on the new state. |
| T-07-02-02 | Tampering | Skill autonomy patch optimistic UI race | mitigate | usePatchSkillAutonomy onMutate snapshot + onError rollback (RESEARCH §Pattern 2); on settle invalidates qk.skills() so server truth wins. Backend regex+dotdot rejects malformed names with 400. |
| T-07-02-03 | Information Disclosure | Inline display of decision question / inbox message | accept | Trust boundary is single-user single-machine (PITFALLS.md L256); browser already trusted. |
| T-07-02-04 | Tampering | Path-bound skill name in /api/skills/{name}/autonomy | mitigate | Frontend uses encodeURIComponent (RESEARCH §Don't Hand-Roll); backend regex `^[a-zA-Z0-9_-]+$` + `..` substring rejection (Plan 03-05 STATE.md L153). |
</threat_model>
