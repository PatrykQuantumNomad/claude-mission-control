<decisions>
- **Cmd+K shape:** Option B (context-aware "Compare with…" + simple navigate when no `a`). Locked per CMPR-03 wording. *Rationale:* directly satisfies the "Compare with…" copy in the requirement. Implementation: use `useRouterState({ select: (s) => s.location })` to detect current route. If on `/sessions/compare?a=X` and `b` is empty → label is "Compare with…" and clicking opens a sessions sub-list (reuse `useSessionsList({ range: '7d', limit: 50 })`). If no current `a` → label is "Compare sessions" and navigates to `/sessions/compare` with empty params (page renders empty-state prompting user to pick).
- **Sessions-table row action:** Add a 7th column `id: 'actions'` whose `cell` renders a `<Button variant="ghost" size="sm">Compare</Button>`. Click navigates to `/sessions/compare?a={r.session_id}`. Do NOT add a generic `onRowClick` to the `DataTable` primitive — out of scope; SkillRunsTable.tsx:11-15 deliberately defers this.
- **Self-compare guard (defensive):** When the user is on `/sessions/compare?a=X` and clicks "Compare" on a row whose `session_id === X`, navigate sets `b` to the *other* of the two values, OR the button is disabled — pick whichever is simpler (likely: just disable the button when `row.session_id === currentA`). Backend already rejects 400 (Plan 16-01), but the UI shouldn't even let the user try.
- **Picker drawer (Sheet) for Cmd+K Option B's session-list sub-step:** Use the existing `ui/Sheet.tsx` primitive. Reuse `useSessionsList({ range: '7d', limit: 50 })` from `lib/queries.ts:183`. *Rationale:* drawer pattern matches `SkillRunsTable.tsx:273-280` precedent.
- **Button label policy:** Match research wording. Cmd+K item label: "Compare with…" (when context-aware) OR "Compare sessions" (when no current `a`). SessionsTable button text: "Compare".
</decisions>
