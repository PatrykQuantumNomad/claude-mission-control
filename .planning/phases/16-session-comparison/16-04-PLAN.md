  <what-built>
    Phase 16 Session Comparison feature end-to-end:
    - Backend: `GET /api/sessions/compare?a=&b=` returning paired-metrics payload (skill-set diff, tool counts, token totals, computed cost via cmc.pricing, outcome, duration, over_cap flag)
    - Frontend: `/sessions/compare?a=&b=` deep-linkable file-based route with hand-written validateSearch UUID validator
    - Two-up `SessionCompareView` panel with KPI strip, side-by-side recharts BarChart, skill-set diff (Shared / Only A / Only B), tool-counts diff table, over-cap EmptyState fallback
    - Cmd+K "Compare sessions" / context-aware "Compare with…" action with Sheet drawer session picker
    - SessionsTable per-row "Compare" button (7th column) navigating to compare with `a` set
  </what-built>
  <how-to-verify>
    **Pre-req:** ensure both servers are running (`pnpm dev` in `frontend/`, `uv run uvicorn cmc.app.factory:create_app --factory --reload --port 8001` in `backend/`). Ingest some sessions if dev DB is empty.

    **1. Empty state (CMPR-02 deep-linkable URL state):**
    - Navigate to `http://localhost:5173/sessions/compare` (no params)
    - Expect: page title "Session Compare", empty state with copy "Pick two sessions from /activity or via Cmd+K", no console errors
    - Navigate to `http://localhost:5173/sessions/compare?a=zzz&b=zzz` (invalid UUIDs)
    - Expect: same empty state — `validateSearch` strips invalid UUIDs to undefined

    **2. Populated state (CMPR-01, CMPR-02):**
    - From `/activity`, click the new "Compare" button on a session row
    - Expect: navigates to `/sessions/compare?a={uuid}` and shows side A populated, side B empty (prompt to pick)
    - Open Cmd+K (Cmd-K) — expect to see "Compare with…" item in Actions group
    - Click it — expect a Sheet drawer with recent sessions
    - Click a different session in the drawer — expect URL updates to include `?a=X&b=Y` and the two-up layout renders fully
    - Confirm KPI strip shows: cost (as `$X.XXXX` Decimal-string template, NOT truncated), duration, tokens (input/output/cache_read/cache_create_5m/cache_create_1h), tool_call_count, message_count, model, outcome `<StatePill>`
    - Confirm side-by-side recharts BarChart renders with explicit height (no 0px collapse)
    - Confirm skill-set diff section shows three columns: "Shared", "Only A", "Only B" — each skill name links to `/skills/$name`
    - Confirm tool-counts diff table shows tool_name + count_a + count_b + delta rows

    **3. Over-cap fallback (CMPR-04):**
    - Find or seed a session with `tool_call_count > 500` (or use a SQL update for testing: `UPDATE sessions SET tool_call_count = 501 WHERE session_id = '<sid>'`)
    - Navigate to `/sessions/compare?a={over-cap-sid}&b={normal-sid}`
    - Expect: KPI strip + skill diff still render for both sides; tool-counts table for over-cap side shows EmptyState with copy "Session too long for full diff (>500 tool calls). Showing summary metrics only."
    - Open browser devtools Network tab, hit the endpoint directly: `curl 'http://localhost:8001/api/sessions/compare?a={over-cap-sid}&b={normal-sid}' | jq`
    - Expect: HTTP 200, `over_cap: true` at top level, side `a.over_cap: true`, `a.tool_counts: {}`

    **4. Picker entry points (CMPR-03):**
    - From `/activity`, click Compare on any row → URL updates to `?a={sid}` ✓
    - From `/sessions/compare?a=X`, hit Cmd+K → "Compare with…" appears (context-aware) ✓
    - From `/sessions/compare` with no params, hit Cmd+K → "Compare sessions" item appears (default case) ✓
    - Self-compare guard: in the Cmd+K Sheet picker, hover over the row matching `a` — expect button disabled or row not selectable

    **5. Structured-tabular-only constraint (CMPR-05):**
    - In browser devtools Network tab, confirm the `/api/sessions/compare` response contains NO LLM message content (only structured numerics + skill-name strings)
    - Confirm no `react-diff-viewer` / `jsdiff` / `diff` JS chunks load (devtools Sources tab — search loaded modules)
    - Run from repo root: `grep -r "react-diff-viewer\|jsdiff\|from ['\"]diff['\"]" frontend/src` — must return NO matches

    **6. Error shapes:**
    - `curl 'http://localhost:8001/api/sessions/compare?a=zzz&b={valid}'` → 400 "invalid session_id format"
    - `curl 'http://localhost:8001/api/sessions/compare?a={non-existent}&b={valid}'` → 404 "session not found"
    - `curl 'http://localhost:8001/api/sessions/compare?a={x}&b={x}'` → 400 "cannot compare a session with itself"

    **7. Decimal precision (Pitfall 1 / CMPR-01):**
    - Inspect a `cost_usd` value in the JSON response
    - Confirm it's a JSON STRING (e.g., `"cost_usd": "0.0247"`), not a number `0.0247`
    - In the rendered KPI strip, confirm the displayed dollar figure matches the string verbatim (no precision loss from `Number(...)` cast)

    **8. Lighthouse / accessibility quick check (optional but recommended):**
    - Run Lighthouse on `/sessions/compare?a=X&b=Y` — expect a11y score ≥ 90 (mirror project bar)
