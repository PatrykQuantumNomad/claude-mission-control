The 9 open questions from `16-RESEARCH.md` resolved here so the implementer has zero ambiguity:

1. **Route file location (FE — informs server contract):** `frontend/src/routes/sessions_.compare.tsx` (trailing-underscore parent-layout opt-out). Locked here because the URL is `/sessions/compare` regardless; backend just needs to know the URL path that consumes it. *Rationale:* mirrors the only existing dynamic-route precedent `skills_.$name.tsx`; future-proofs against a future `routes/sessions.tsx` parent.
2. **Cmd+K shape:** Option B (context-aware "Compare with…"). Decided in Plan 16-03; no impact on this plan.
3. **Tool counts shape:** Full dict of every distinct `tool_name` for the side (no top-N truncation). *Rationale:* dict size bounded by ≤500 tool calls and ≤30 distinct tool names; truncation adds churn for no payload-size win.
4. **`rates_as_of` granularity:** Single top-level field = `max(effective_from of model_a rate, effective_from of model_b rate)`. *Rationale:* one timestamp is enough for the audit footer; per-side adds noise.
5. **`cost_usd` JSON shape:** Decimal-string (Pydantic v2 default for `Decimal`). *Rationale:* Phase 13/14 lock; frontend MUST template-literal display, NEVER `Number(...)` (precision loss — Pitfall 1 in research).
6. **Sessions-table picker UX:** Sheet drawer (Plan 16-03 detail). Backend is unaffected.
7. **Self-compare (`a == b`):** Reject 400 server-side with `{"detail": "cannot compare a session with itself"}`. Frontend will also disable the button (Plan 16-03). *Rationale:* degenerate UX, all metrics identical, empty diff — confusing.
8. **NavBar entry:** No NavBar link (contextual route). Plan 16-02 detail; backend unaffected.
9. **Cost-module path drift in REQUIREMENTS/ROADMAP (`cmc/cost/engine.py` does NOT exist):** Use the actual `cmc.pricing` module — the codebase truth — and add a one-line note in the Plan 16-04 SUMMARY pointing to this deviation. Phase 17 docs scope can clean up REQUIREMENTS/ROADMAP text. *Rationale:* code path is the source of truth; do not synthesize a shim.

**Over-cap status code:** 200 with `over_cap=true` + `tool_counts={}` (NOT 413/422). The roadmap says "show summary metrics fallback" — that's a render branch, not an error.

    ```python
    @sessions_router.get("/compare", response_model=SessionCompareResponse)
    async def compare_sessions(
        a: str,
        b: str,
        db: AsyncSession = Depends(get_db_session),
    ) -> SessionCompareResponse:
        # 1. Validate both UUIDs via _UUID_RE; reject 400 if malformed (mirror sessions.py:111-112)
        # 2. If a == b: raise HTTPException(400, "cannot compare a session with itself")
        # 3. Load each session row by session_id (mirror sessions.py:118 for 404 on miss)
        # 4. rates = await load_rates(db) — single call, shared across both sides
        # 5. For each side: build a SessionCompareSide via _build_side(sess, rates, db)
        # 6. Compute skill_diff = SkillSetDiff(shared=sorted(set_a & set_b), only_a=sorted(set_a - set_b), only_b=sorted(set_b - set_a))
        # 7. rates_as_of = max effective_from across the two models actually used (or None if unknown)
        # 8. Return SessionCompareResponse(a=side_a, b=side_b, skill_diff=skill_diff, rates_as_of=..., over_cap=side_a.over_cap or side_b.over_cap, cap=SESSION_COMPARE_CAP)
    ```
