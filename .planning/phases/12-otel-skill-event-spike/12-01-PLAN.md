OTEL body envelope (CRITICAL — Pitfall 2):
- Schema: `{"record": {"attributes": [{"key": "...", "value": {"stringValue": "..."}}, ...], "body": {"stringValue": "claude_code.<event>"}}, "resource": {...}, "scope": {"name": "...", "version": "..."}}`
- Attributes are an ARRAY of `{key, value}` objects, NOT a flat dict
- ALWAYS use `json_each(json_extract(body, '$.record.attributes'))` and filter `$.key`, then read typed value off `$.value.stringValue` / `$.value.intValue`
- NEVER use bare `json_extract(body, '$.attr_name')` for OTLP attributes — that pattern silently returns NULL (this is the v1.0 latent bug at `cmc/api/routes/observability.py:535`)

- The full prefixed name (`claude_code.skill_activated`) lives at `body.record.body.stringValue`
- BOTH should be captured in the raw appendix so Plan 02 can lock both surfaces
