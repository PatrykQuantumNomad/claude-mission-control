<!-- AlertRulesList behavior: -->

PanelCard wraps useAlertRules() — handles skeleton + error + empty.
DataTable columns:
  rule_id (numeric, narrow)
  name (string)
  kind (Badge: threshold | anomaly)
  metric (truncated string)
  threshold_fire (number; '—' for anomaly with null)
  enabled (toggle switch — usePatchAlertRule({id, body: {enabled: !current}}))
  actions (Delete button — useDeleteAlertRule(id) with confirm)
Empty: EmptyState 'No alert rules yet — fill the form to create your first.'

<!-- AlertRuleForm behavior: -->

useCreateAlertRule hook + local form state.
Segmented control (Threshold / Anomaly) drives field rendering:
  Both: name (string), metric (dropdown of KNOWN_METRICS), enabled (default true).
  Threshold: threshold_fire (number, required), threshold_clear (number, optional),
             min_dwell_seconds (number, default 0), cooldown_seconds (number, default 0).
  Anomaly: threshold_fire (z-score, default 3.0), threshold_clear (default 1.5),
           min_samples (number, default 10), min_dwell_seconds, cooldown_seconds,
           params_json.window_n (number, default 50).
Client-side validation (mirrors backend validators):
  - name length 1-120
  - threshold_clear < threshold_fire when both set
  - threshold rules require threshold_fire
On submit: useCreateAlertRule.mutate(body); on success → reset form + show success toast; on 422 → show error toast with detail.

<!-- AlertEventsList behavior: -->

useAlertEvents(range) where range comes from RangeToggle with persistKey='alert-events-range'.
DataTable columns:
  fired_at (RelativeTime)
  rule_name
  scope_key (truncated)
  status (StatePill: pending = warning, answered = success)
  last_value (number formatted with 2-decimal)
Empty: EmptyState 'No alerts have fired in this range.'
