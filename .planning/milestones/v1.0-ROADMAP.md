# Milestone v1.0: MVP — Claude Mission Control

**Status:** ✅ SHIPPED 2026-04-28
**Phases:** 1–11 (9 base + 2 gap closure)
**Total Plans:** 47

## Overview

A production-grade local dashboard and command centre for Claude Code. Ingests session JSONLs and OTEL telemetry from the user's Mac, stores everything in SQLite (WAL), and renders a dense, dark-themed React UI at `localhost:8765`. Includes a task dispatcher (Mission Control), human-in-the-loop decisions/inbox, scheduled task automation, live session monitoring, and an optional Telegram pager. Runs entirely on localhost — no cloud, no accounts, no outbound telemetry.

## Milestone Summary

**Decimal Phases:** None (Phases 10–11 are integer gap-closure phases appended after Phase 9)

**Key Decisions:**

- **SQLAlchemy 2.0 async + SQLModel + Alembic** (locked in Phase 1 discussion) — Models give type-safe queries, Alembic gives versioned migrations, aiosqlite stays as the async driver. Supersedes earlier "raw SQL" direction in REQUIREMENTS.md (FOUND-02/03 wording is intent, not implementation).
- **FastAPI chassis patterns** (app factory, builder, settings, lifespan) — User's established production patterns for consistency and quality.
- **Python 3.13** — User preference for latest Python; chassis targets 3.13+.
- **SQLite single-file with WAL** — Local-only, no multi-server, excellent read concurrency for dashboard workload.
- **PID files for emergency stop targeting** — macOS 12+ restricts env disclosure to root; PID files are the only reliable way to identify dispatched children.
- **TanStack Router (file-based) + React Query (30s poll)** — Modern React routing + efficient data fetching without WebSocket complexity. Locked polling cadences in lib/queries.ts.
- **Launchd for daemon management** — macOS-native process supervision, no third-party dependency.
- **SSE for firehose, polling everywhere else** — Simpler than WebSockets, sufficient for single-user local dashboard.
- **Plain text Telegram messages (no parse_mode)** — DB-sourced content has unescaped backticks that break markdown parsing silently. Pitfall P3 enforced by inspect.signature() grep test.
- **Theme toggle locked to Q1=A in Phase 9** — Light mode CSS overrides + lib/theme.ts module + NavBar mount.
- **cmc CLI shim (renamed from cc)** — Avoids /usr/bin/cc (clang) collision discovered during Phase 9 install testing.

**Issues Resolved:**

- Telegram Reject button → 404 (Phase 10 added POST /api/tasks/{id}/reject)
- answered_by mis-tag for Telegram-sourced answers (Phase 10 stamps answered_by='telegram' in dash_router.py:58-63)
- REQUIREMENTS.md INGST-02/03/05/06/08 stale traceability rows (Phase 11 flipped to Complete)
- doctor.py false-skipping Telegram check (Phase 11 added .env loader)
- TELE-05 ANTHROPIC_API_KEY surfaced via Settings rather than bare os.environ (Phase 11)
- Notifier inbox loop refactored to call /api/inbox/{id}/read (HTTP symmetry, Phase 11)
- Phase 9 install bugs: cmc restart, telegram .env parsing, frontend cc→cmc strings, daemon start on macOS

**Issues Deferred to v2:**

- ACTV-04 TopSkills functional panel — needs claude_code.skill_invoked OTEL event in ingest pipeline

- No frontend useRejectTask hook (reject is Telegram-only by Phase 10 design; dashboard rejects use Delete which sets status=removed; cross-channel terminal-state parity deferred)
