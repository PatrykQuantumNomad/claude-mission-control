## Milestone Summary

**Decimal Phases:** None — all 6 phases were planned base phases (no urgent insertions).

**Key Decisions:**

- **Cost stored as tokens, $ computed at read time** — pricing changes self-correct historical totals via `effective_from`/`effective_until` window; `/v1/logs` MUST always return 200 (Pitfall 4); never store $ in derived tables
- **Single Alembic migration 0002** — `attrs_skill_name` + alert tables + cache TTL split + UNIQUE constraint + BUG-B backfill in one change to avoid mid-milestone migration ordering issues
- **Alerts emit decisions only** (ALRT-12) — alert engine NEVER imports `cmc.dispatcher.tasks`; user gates action via existing autonomy controls; "alerts are sensors, not actuators"
- **Stable `dedup_key = f"alert:{rule_id}:{scope_key}"`** — reuses `notification_log` UNIQUE(kind, entity_id, chat_id) + `decisions` partial-unique on dedup_key WHERE status='pending'; no new state machine
- **`cmc/telegram/callback_verbs.py` central StrEnum** — refactored `dash_router.py` + `handler.py` to dispatch via enum; sha256[:8] encoding stays under 64-byte Telegram callback_data cap
- **`UTCDatetime = Annotated[datetime, PlainSerializer(_serialize_utc, when_used='json')]`** — project-wide drop-in for every Pydantic v2 schema datetime field; `when_used='json'` gate keeps native datetime objects for SQLAlchemy INSERT/UPDATE
- **Hand-written `validateSearch` UUID validator** — first use in the codebase; deliberately avoided pulling in zod/valibot for one regex check; `routes/sessions_.compare.tsx` sets the precedent
- **Tabular-only session compare** (CMPR-05) — no react-diff-viewer / jsdiff; metadata only; full message diff is privacy-leaky and visually overwhelming
- **CMPR-04 over-cap = render branch, not error branch** — HTTP 200 + `over_cap=true` + empty `tool_counts={}` so client templates an EmptyState while KPIs survive
- **TanStack parent-layout opt-out via trailing-underscore filenames** — `skills_.$name.tsx` and `routes/sessions_.compare.tsx` keep public URLs unchanged while bypassing parent route's missing `<Outlet/>`
- **Wave-1/wave-2 single-writer convention** — Phase 16-04 + 17-06 serialize REQUIREMENTS.md edits behind on-disk verified artifacts from parallel wave-1 plans

**Issues Resolved:**
